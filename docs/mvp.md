# MVP de Complienx

## Objetivo

Crear una primera versión funcional enfocada en control documental, aprobaciones y acciones de cumplimiento.

## Módulos iniciales

### 1. Dashboard

Vista general del estado del sistema:

- Documentos activos.
- Documentos pendientes de aprobación.
- Acciones abiertas.
- Tareas vencidas.
- Actividad reciente.
- Indicador básico de cumplimiento documental.

### 2. Documentos

Funciones iniciales:

- Crear documento.
- Subir archivo.
- Clasificar por categoría.
- Asignar responsable.
- Definir estado.
- Consultar historial.
- Controlar versiones.

Estados sugeridos:

- Borrador.
- En revisión.
- Aprobado.
- Rechazado.
- Vencido.
- Archivado.

### 3. Aprobaciones

Funciones iniciales:

- Ver documentos pendientes.
- Aprobar documento.
- Rechazar documento.
- Agregar comentario de decisión.
- Registrar usuario, fecha y resultado.

### 4. Acciones

Este módulo agrupa acciones correctivas, preventivas y de mejora.

Funciones iniciales:

- Crear acción.
- Definir tipo.
- Asignar responsable.
- Definir prioridad.
- Definir fecha compromiso.
- Cambiar estado.
- Agregar comentarios de seguimiento.

Tipos sugeridos:

- Correctiva.
- Preventiva.
- Mejora.

Estados sugeridos:

- Abierta.
- En proceso.
- En revisión.
- Cerrada.
- Cancelada.

### 5. Tareas

Bandeja de pendientes para cada usuario.

Tipos de tareas:

- Documento por aprobar.
- Acción por atender.
- Documento próximo a vencer.
- Acción vencida.

### 6. Usuarios y roles

Roles iniciales:

- Administrador.
- Colaborador.
- Auditor o consulta.

## Fuera del MVP inicial

Se dejan para fases posteriores:

- Encuestas.
- Resultados de encuestas.
- Calibración.
- Análisis avanzado de riesgos.
- Evolución de riesgos.
- Proyectos de mejora como módulo independiente.
- Reportes PDF avanzados.
- Automatizaciones complejas.
- Integraciones externas.
